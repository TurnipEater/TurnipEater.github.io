One solution to the problem of adding a navigation bar would be if H5P had an activity called, well shall we call it, WebPage. Yes, there are the text panels that can be added to the column activity and some other places, but the styling options are very limited and, presumably, intentionally locked down. Perhaps there is a technical reason for this but I find this doubtful. I cannot imagine there is a technical issue that stops a user from choosing a font they find more appropriate for their particular setting. If it's not a technical issue, I can only think it is part of the philosophy of H5P, maybe to preserve a particular look and feel. If this is so, then I'm not persuaded. Far better, I would have thought, to let the end user decide for themselves what looks best when an activity has been embedded in wherever they are using it. Likewise in terms of what users should be able to put into that web page.

Admins may have some freedom to add extra buttons to the various editor but that doesn't really help the majority of us who just have to use things 'out of the box'.

There will undoubtedly be security issues that need to be considered and, presumably, scripts would still need to be stripped out. However, I see little danger in allowing the user access to all the styling options or the inclusion of images in with the text. Allowing users to be creative may lead to all sorts of activities that may in time be developed and adopted in a more official way.

It just seems to me to be very puzzling that H5P stands for HTML5 Package but the normal user is never given the option of editing the HTML. True everything is made available on an open-source basis and there is nothing to stop people writing their own activity module for their own setup or paying a developer to do so, but this sets the bar very high. There will, undoubtedly, be lots of users who could use a more comprehensive editor and even edit the HTML directly who are not able to create a brand-new module.

Even the use of iFrames seems to be discouraged. I do understand that it can be difficult, and sometimes maybe impossible, to make iFrames properly responsive but, once again, I would argue it is up to the person compiling the activity for their learners to decide if and when it works for them.

Oh well, H5P is still a phenomenal set of activities despite any frustrations I might have.
